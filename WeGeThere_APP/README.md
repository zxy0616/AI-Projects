## 1. 项目概述

### 1.1 项目背景
解决用户在出行中面临的“盲目搜索、异地聚合难、攻略复杂、社交不靠谱”四大痛点 。

### 1.2 产品定位
**WeGethere (集游)**：一款以 AI Agent 为核心的多人智能出行规划与社交协作 APP。

---

## 2. 用户画像与场景

### 2.1 核心用户画像
1. **盲目探索者**：想出游但无明确目的地，厌恶繁杂的社交媒体推荐 。
2. **异地聚合群**：多位好友分布于同城不同区域，对集合点的公平性与便捷度有强需求。
3. **多人旅行规划困难者 (新增)**：长途旅行目标筛选耗时、难以均衡各方喜好、订票定酒店等全过程攻略过于繁琐 。
4. **社交寻求者**：有目的地但无同伴，对网络组队的安全性高度敏感。

### 2.2 典型场景
* **场景 A (复杂决策)**：用户A 与 用户B 计划长途跨省游，涉及多个备选城市筛选、机酒比价与喜好权衡。
* **场景 B (AI 介入)**：群聊中成员对行程产生分歧，AI 在监测到 3 秒静默后主动切入提供优化建议。

---

## 3. 详细功能逻辑

### 3.1 AI 智能协作与权限边界
* **数据驱动规划**：整合主流社媒信息并剔除广告，根据用户喜好（看展、美食等）及出行模式（紧凑/悠闲）智能推荐 。
* **AI 交互边界**：
    * **被动响应**：支持用户通过 `@AI 助手` 获得即时方案修改建议。
    * **主动提醒 (Active-Trigger)**：系统通过监测群聊输入状态并分析上下文，若 3 秒内无成员输入且无“正在输入”状态，AI 将主动插入建议（如：拥堵提醒、优化顺序建议等） 。

### 3.2 集合点规划与多分支同步
* **集合点选项**：提供不限、路程相同、时间相同、距某一成员较近四种算法模型。
* **分支逻辑 (Itinerary Branching)**：
    * **同步机制**：分裂后的多组人可互看彼此的规划方案。
    * **隐私保护**：实时位置共享默认为关闭状态，用户可根据需求自行选择开启。

### 3.3 游伴招募与安全验证
* **准入制度**：必须通过实名认证与人脸核验。
* **信用体系**：包含信誉标签、特长说明（摄影、驾驶等）及行程后的匿名互评。

### 3.4 支付集成与“团队小金库” (核心更新)
* **资金存储**：保证金与充值资金均冻结在 APP 内。
* **支付优先级**：
    1.  **一类资金 (自由金)**：成员额外投入的资金，用于日常集体消费（如聚餐）。
    2.  **二类资金 (保证金)**：作为团队支出的保底金，仅在自由金不足或产生异常违约时调用。
* **支出与分摊逻辑**：
    * **一键买单**：支持买单者上传凭证并选择涉及人数。
    * **强制执行**：若成员拒绝分摊，系统将自动从其保证金中扣除相应金额。
    * **异议仲裁**：成员可对异常支出提出异议，流程为：`团队内部表决` -> `平台介入责任划分`。

## 4. 交互流程定义

### 4.1 核心演示路径
1.  **入口**：用户登录（示例用户名为Evelyn）。
2.  **探索**：首页查看推荐流 -> 进入帖子详情 -> 添加至意向单。
3.  **创建**：首页点击“新建日程” -> 填写偏好及意向地点。
4.  **招募**：一键发布招募贴 -> 跳转游友圈查看。
5.  **管理**：进入招募管理查看申请人（Emma, Kitty） -> 开启群聊。
6.  **协作**：AI 主动/被动介入生成“智能集合点”及“详细日程方案”。
7.  **执行**：首页显示实时行程提醒，展示已确定的规划详情。

---

## 5. 非功能性需求

* **交易安全**：所有小金库支出均需保留电子凭证，异常扣款需由平台人工介入二次审核。
* **隐私规范**：位置共享需明确告知权限有效期，到期自动关闭。
* **计算效率**：多地集合的最优解计算耗时需控制在 3 秒以内。

---

## 6. 核心结算公式

$$最终退还金额 = 保证金 + 剩余自由金 - (\text{个人分摊损益} + \text{仲裁判定赔付})$$
